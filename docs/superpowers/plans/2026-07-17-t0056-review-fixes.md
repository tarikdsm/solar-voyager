# T0056 Review Fixes Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Resolve every Critical, Important, and Minor finding from the independent T0056 review without weakening the v1 HUD contract.

**Architecture:** Publish an allocation-free active-or-latest burn summary through four primitive `SimSnapshot` fields governed by ADR-028, then sample and format those fields in the existing HUD signal graph. Use a dimension-aware stacked layout and browser regressions to prevent intermediate-width collisions.

**Tech Stack:** TypeScript strict, Vitest, Preact signals, CSS, Playwright Chromium, Vite.

## Global Constraints

- `src/sim/` remains pure TypeScript and all new sim behavior has Vitest coverage.
- `SimSnapshot` changes require an ADR and use only preallocated primitive storage.
- Energy/power formatting continues through the shared core formatters.
- No frame-loop literals, closures, objects, arrays, or material/geometry creation.
- UI numeric telemetry stays sampled at 10 Hz; warp clamp state remains same-frame.
